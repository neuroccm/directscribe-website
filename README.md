# directscribe.ca

Marketing + compliance website for **DirectScribe** — a Mac-native,
bring-your-own-API-key medical dictation app for solo outpatient primary-care
physicians in Canada. Implements units **U13** and **U14** of the
commercialization plan.

## Stack

Hand-authored static **HTML + CSS**, generated by a small zero-dependency
Python script. Astro was the preferred stack, but `node`/`npm` were unavailable
in the build environment, so the site is plain static output — still fast,
still deploys to Cloudflare Pages, and ships **zero client JavaScript**.

- `src/layout.html` — the one shared shell: `<head>`, persistent nav, footer,
  and the global NOT-LEGAL-ADVICE disclaimer. Every page inherits it, so the
  chrome and disclaimers can never drift between pages.
- `src/pages/**/*.html` — per-page content fragments. A leading HTML comment
  carries `title`, `description`, and `nav` (which top-nav item to highlight).
- `src/styles.css` — the whole design system (blue/white/silver, WCAG AA).
- `public/**` — copied verbatim to the site root (`version.json`, `favicon.svg`,
  `_headers`, and `screenshots/`).
- `build.py` — stamps each fragment into the layout and writes clean-URL static
  files to `dist/`.
- `validate.py` — the build gate (see below).
- `dist/` — the generated site. **Committed on purpose** so Cloudflare Pages can
  serve it with no build step. Never edit it by hand; edit `src/` and rebuild.

Clean URLs come from the directory/`index.html` pattern, so the app's deep links
(`directscribe.ca/vendors/deepgram`, etc.) resolve exactly.

## Preview locally

```sh
python3 build.py                       # regenerate dist/
python3 validate.py                    # link + copy + a11y checks (must pass)
python3 -m http.server -d dist 8000    # serve it
# open http://localhost:8000
```

`validate.py` fails the build on: a broken internal link or anchor, a missing
`<h1>`/`<title>`/lang/viewport, a missing nav or footer, any `<script>` tag or
inline style/handler (the site ships no JS), a compliance page without an
in-body NOT-LEGAL-ADVICE callout, any honest-copy over-claim from the denylist
(e.g. "100% local", "nothing retained anywhere", "HIPAA compliant"), a missing
required phrase, a malformed `version.json`, or a colour pair below WCAG AA.

## Deploy to Cloudflare Pages

**Option A — no build step (simplest).** Because `dist/` is committed:

1. Cloudflare dashboard → Workers & Pages → Create → Pages → Connect to Git.
2. Pick this repo.
3. Framework preset: **None**. Build command: **(leave empty)**.
   Build output directory: **`dist`**.
4. Save and Deploy. Add the custom domain `directscribe.ca`.

**Option B — build on Cloudflare (keeps `dist/` fresh from source).**
Same as above, but set Build command: **`python3 build.py`** (the Pages build
image includes Python 3). Output directory: **`dist`**.

Either way, `public/_headers` applies the security headers (CSP with
`script-src 'none'`, HSTS, `X-Content-Type-Options`, etc.) and caching.

## Content review cadence

Vendor pricing and endpoints in `src/pages/cost.html` and the vendor guides were
verified **2026-07-02**. Prices drift — review these on the app's monthly cadence
and update the figures. Anthropic small-customer BAA copy is deliberately

resolved.

## Honest-copy constraints (enforced by `validate.py`)

These were explicit review findings and must not regress:

- "Local-first" is precise: at launch **every transcription path is cloud**; only
  LM Studio note-shaping is fully local.
- Deletion copy: "deleted from the file system on your schedule; FileVault
  protects residual storage; backups you configure may retain copies" — never
  "nothing retained anywhere".
- The audit log is a transmission/deletion transparency record — **not** a
  PHIPA s.10.1 access-audit log; the EMR is the system of record.
- Canada-first (PIPEDA + PHIPA), not HIPAA.
- The weekly update check is disclosed alongside "no telemetry", with its off
  switch noted.
